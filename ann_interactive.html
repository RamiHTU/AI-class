Section 02
The Perceptron
The simplest ANN — invented in 1953 by Frank Rosenblatt. It computes a weighted sum of inputs and applies a step function to decide an output.
A perceptron is also called a Linear Threshold Unit (LTU). It computes:
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
then applies a step function: output 1 if z ≥ 0, else output 0.
z = w₁x₁ + w₂x₂ + … + wₙxₙ + b  |  f(z) = 1 if z ≥ 0, else 0
Training a perceptron means finding the right values for the weights w and bias b so the output matches the correct label.

Step Function
Hard threshold: output is exactly 0 or 1. Simple but cannot learn complex, non-linear patterns.

Section 03
Activation Functions
Instead of a hard step, modern networks use smooth activation functions — enabling gradient-based learning and the ability to model complex, non-linear patterns.
📈 Activation Function Plotter
📊
Sigmoid σ(x)
Squashes any value to (0,1). Used in binary classification — output can be interpreted as probability. Formula: 1 / (1 + e⁻ˣ)
⚡
ReLU f(x)
Rectified Linear Unit: max(0, x). Simple and fast — the most popular activation in hidden layers. Introduces non-linearity efficiently.
〰️
Tanh
Outputs between −1 and 1. Zero-centered, which can speed up learning. Often used in recurrent networks.
Why do we need non-linearity? A network made of only linear operations is mathematically equivalent to a single linear transformation — no matter how many layers you stack. Non-linear activations are what allow neural networks to learn complex, curved decision boundaries.

Section 04
Multilayer Perceptrons
Stack multiple layers of neurons — each layer transforms the representation, allowing the network to learn increasingly abstract patterns.
🏗️ MLP Architecture Explorer

Click a neuron to highlight its connections. Hover to see its layer type.

📥
Input Layer
Neurons = number of features. Receives raw data, no computation — just passes values forward.
🔮
Hidden Layer(s)
Where the magic happens. Each neuron computes z = Σ(wᵢxᵢ) + b, then applies an activation function. Contains bias.
📤
Output Layer
Neurons = number of classes. Uses sigmoid (binary) or softmax (multi-class) to produce final prediction.
Feedforward networks: information flows in only one direction — left to right (input → hidden → output). This is also called a directed acyclic graph. Each neuron in layer L is connected to every neuron in layer L+1. This is called a fully connected or dense layer.

Section 05
Backpropagation
The algorithm that makes deep learning possible — efficiently computing how much each weight contributed to the error, then adjusting them all at once.
1
Forward Pass — Feed input data through the network. Each layer computes z = Wx + b, then applies activation f(z). Store all intermediate values.
2
Error Calculation — Compare the network's prediction ŷ with the true label y. Compute loss: e.g., MSE = (y − ŷ)² or Cross-Entropy for classification.
3
Backward Pass — Use the chain rule to compute ∂Loss/∂w for every weight. Gradients flow from the output layer back through each hidden layer — reusing partial results efficiently.
4
Weight Update — Adjust each weight: w ← w − α × ∂Loss/∂w, where α is the learning rate.

Why is backprop efficient? Instead of computing the gradient of each weight separately (which would require a full forward pass per weight), backprop reuses gradients computed in later layers when computing earlier ones — this is the chain rule in action.

Section 06
Optimizers
Backprop tells us the gradient — but how we use that gradient to update weights is the optimizer's job. Different strategies lead to very different training behavior.
Plain Gradient Descent updates every weight as: w ← w − α·∂L/∂w. It works, but can be slow and get stuck. Optimizers are smarter strategies for navigating the loss landscape.
⚙️ Optimizer Comparison — Loss Curve Simulation

Select an optimizer to see how training loss evolves. Notice the different convergence behaviors.

🐢
SGD
Stochastic Gradient Descent — updates weights using one sample (or mini-batch) at a time. Simple, but noisy and slow to converge.
🏃
Momentum
Adds a "memory" of past gradients — like a ball rolling downhill. Accelerates through consistent directions, dampens oscillations.
⚡
Adam
Adaptive Moment Estimation — combines momentum AND adaptive learning rates per parameter. The go-to optimizer for most modern networks.
📐
RMSProp
Adapts the learning rate for each weight based on recent gradient magnitudes. Effective for recurrent networks and noisy gradients.
Analogy: Imagine you're blindfolded on a hilly landscape trying to find the lowest point. SGD takes random steps. Momentum remembers which direction you were going. Adam watches how steep each step has been and adjusts stride length per direction. That's why Adam usually wins.
