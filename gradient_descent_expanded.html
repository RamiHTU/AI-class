01
The Big Picture

The Mountain Analogy

Imagine you are standing somewhere on a foggy mountain at night. Your goal is to reach the lowest valley. You can't see far ahead — the fog limits your view to just a few metres. What do you do?

You feel the slope beneath your feet and take a small, careful step downhill. Then you reassess. Then step again. You repeat this until the ground feels flat beneath you — you've reached the valley floor.

Gradient descent in one sentence: repeatedly take small steps in the direction that reduces the cost function, until you can't reduce it further.

Why Do We Need This at All?

In machine learning, we want to find the model parameters (weights) that make our predictions as accurate as possible. This means minimising a cost function — a number that tells us how wrong our model is.

For simple problems with one or two parameters, we could find the minimum analytically (set the derivative to zero and solve). But real models have thousands or millions of parameters. Analytical solutions become computationally impossible. We need an iterative, automated approach — that's gradient descent.

A Brief History

Gradient descent was first proposed by Augustin-Louis Cauchy in 1847. It remained largely theoretical for over a century. With the rise of neural networks in the 1980s and the backpropagation algorithm, it became the dominant method for training machine learning models. Today it powers GPT, image recognition, recommendation systems — virtually every AI system you interact with.

02
What We Are Minimising

The Cost Function

Before we can descend, we need something to descend on. The cost function J(θ) — also called the loss function or objective function — measures how wrong our model's predictions are across the entire training set. It converts every error into a single number.

Why a Single Number?

Our model makes one prediction per training example. If we have 1000 training examples, we have 1000 errors. Gradient descent needs a single value to minimise. The cost function aggregates all those individual errors into one score we can optimise.

Mean Squared Error — The Standard Choice

For regression problems, Mean Squared Error (MSE) is the most common cost function:

J(θ) = (1 / 2n) · Σᵢ (ŷᵢ − yᵢ)²

Symbols Explained

SymbolMeaning
nNumber of training examples
ŷᵢModel's predicted value for example i
yᵢActual (true) value for example i
(ŷ−y)²Squared error — always positive
½Convenience factor — cancels with the 2 that appears when we differentiate

Why Squared?

Squaring the error does two things: it makes all errors positive (so negative and positive errors don't cancel out), and it penalises large errors more heavily than small ones — a prediction that's 10 units off contributes 100× more to the cost than one that's 1 unit off.

The Cost Surface

When plotted against the parameters, J(θ) forms a surface in space. For linear regression with one parameter, it's a simple curve. With two parameters (m and b), it forms a bowl shape — the minimum is at the bottom of the bowl.

The parabola is J(w) = w². The green dot marks the global minimum at w = 0.

Why Not Just Solve for the Minimum Directly?

For linear regression with a small dataset, you can solve analytically using the Normal Equation: θ = (XᵀX)⁻¹Xᵀy. But this requires computing a matrix inverse, which scales as O(n³). With hundreds of features, this becomes prohibitively expensive. Gradient descent scales to millions of parameters and is the only practical approach for deep neural networks.

Note on terminology: "Cost function", "loss function", and "objective function" are often used interchangeably, though strictly speaking the loss is per example and the cost is the average over all examples.

03
Direction Finder

The Gradient

The gradient is the multi-dimensional generalisation of the derivative. It tells us the slope of the cost function at our current position — specifically, how much the cost changes when we nudge each parameter slightly.

The First Derivative — A Refresher

For a function of one variable f(w), the derivative f'(w) gives the slope of the tangent line at w:

f'(w) = df/dw = lim[Δw→0] ( f(w + Δw) − f(w) ) / Δw

Positive derivative → function is increasing → we are to the right of the minimum → step left.
Negative derivative → function is decreasing → we are to the left of the minimum → step right.
Either way, subtracting the derivative moves us toward the minimum.

Move your mouse over the curve to see the tangent line and gradient value at each point.

Partial Derivatives — Multiple Parameters

Real models have many parameters. If J depends on two parameters θ₁ and θ₂, we compute a partial derivative for each — holding all others fixed:

∂J/∂θ₁ — how much J changes when θ₁ changes (θ₂ held fixed)
∂J/∂θ₂ — how much J changes when θ₂ changes (θ₁ held fixed)

The Gradient Vector

The gradient ∇J is a vector containing all the partial derivatives. For a function of n parameters:

∇J(θ) = [ ∂J/∂θ₁, ∂J/∂θ₂, …, ∂J/∂θₙ ]

This vector points in the direction of steepest ascent on the cost surface. To minimise the cost, we move in the opposite direction — hence the minus sign in the update rule.

Gradient for MSE with Linear Regression

For J = (1/2n)·Σ(mx + b − y)², the partial derivatives work out to:

∂J/∂m = (1/n) · Σ (ŷᵢ − yᵢ) · xᵢ
∂J/∂b = (1/n) · Σ (ŷᵢ − yᵢ)
Intuition for the formulas above: the partial derivative w.r.t. m is the average of (error × input x). This makes sense — parameters connected to larger inputs have a bigger influence on the output, so their gradient should account for that.

The Core Equation

The Update Rule

Everything in gradient descent collapses into one compact equation, applied once per iteration (also called an epoch when using the full dataset):

θparameter (new value)
←
θparameter (old value)
−
αlearning rate
·
∂J/∂θgradient of cost w.r.t. θ

Reading the Equation

θ (left side)

The new, updated value of our parameter after this iteration. We overwrite the old value.

α · ∂J/∂θ

The step we take. The gradient tells us the direction and the magnitude of the slope. α scales how large the step actually is.

The minus sign

We subtract the gradient because it points uphill. Subtracting it points us downhill — toward lower cost.

Convergence

As we approach the minimum, the gradient becomes smaller and smaller. Steps get tiny — the parameter values stop changing significantly.

Applied to Linear Regression

For ŷ = mx + b, we update both parameters simultaneously at each iteration:

m ← m − α · (1/n) · Σ(ŷᵢ − yᵢ)·xᵢ
b ← b − α · (1/n) · Σ(ŷᵢ − yᵢ)
⚠ Critical: Simultaneous Update

You MUST compute BOTH gradients using the CURRENT values of m and b, then update BOTH at the same time. Do not update m first and then use the new m to compute the gradient for b — this is a common mistake that produces incorrect results.

✓ CORRECT
Dm = gradient using old m, b
Db = gradient using old m, b
m = m − α·Dm
b = b − α·Db
✗ WRONG
Dm = gradient using old m, b
m = m − α·Dm ← update m first!
Db = gradient using NEW m, b ← error
b = b − α·Db

Full Algorithm — Step by Step

  • 1.Initialize parameters to zero (or small random values)m = 0, b = 0, choose α (e.g. 0.01), choose max iterations
  • 2.Compute predictions for all training examplesŷᵢ = m·xᵢ + b for all i
  • 3.Compute the costJ = (1/2n)·Σ(ŷᵢ − yᵢ)²
  • 4.Compute gradients (partial derivatives) using current parametersDm = (1/n)·Σ(ŷᵢ−yᵢ)·xᵢ    Db = (1/n)·Σ(ŷᵢ−yᵢ)
  • 5.Update both parameters simultaneouslym ← m − α·Dm    b ← b − α·Db
  • 6.Check for convergence. If not converged, go back to step 2.

The Critical Hyperparameter

The Learning Rate α

The learning rate α is a small positive number — typically between 0.0001 and 1.0 — that controls how large each parameter update step is. It is arguably the most important hyperparameter in gradient descent. Getting it wrong can completely break training.

Unlike parameters (m, b), which are learned during training, α is a hyperparameter — a design choice made by the engineer before training begins.

Too Small (α ≪)

Steps are tiny. The algorithm converges but takes many thousands of iterations. Computationally wasteful. The cost curve decreases very slowly.

Just Right

Cost decreases smoothly and reaches the minimum in a reasonable number of iterations. The sweet spot varies per problem.

Too Large (α ≫)

Steps overshoot the minimum. The algorithm bounces back and forth — cost may actually increase. In extreme cases, the parameters diverge to infinity.

Practical Guidelines for Choosing α

  • →Start with α = 0.01 and adjust based on the cost curve
  • →Plot cost vs. iteration. A healthy training curve decreases smoothly and flattens near convergence
  • →If cost oscillates or increases, reduce α by a factor of 10
  • →If cost decreases but training takes forever, try increasing α by a factor of 3
  • →Advanced: use learning rate schedules — start high (fast progress), then decay α over time (fine-tune near minimum)

Three Flavours

Variants of Gradient Descent

The original formulation of gradient descent uses all training examples to compute the gradient at each step. But with millions of training examples, this is extremely slow. Three main variants exist, each making a different trade-off between gradient accuracy and computational speed.

The Core Difference

The only thing that changes between variants is how many training examples are used to compute the gradient at each step. Everything else — the update rule, the convergence goal — remains the same.

Why does noise sometimes help? Stochastic and mini-batch gradient descent are "noisier" — their gradient estimates fluctuate. This noise can actually be beneficial: it can help the algorithm escape shallow local minima that batch gradient descent would get stuck in, potentially finding better solutions.

What is a "Batch Size"?

In mini-batch GD, you divide your training data into small chunks called batches. One pass through all batches (covering the entire training set) is called an epoch. With 1000 examples and a batch size of 100, you perform 10 gradient updates per epoch. Common batch sizes: 32, 64, 128, 256 — powers of 2 work well on GPUs.

Real-World Complications

Challenges of Gradient Descent

Gradient descent is powerful, but it comes with several challenges that practitioners must understand and work around. These are not just theoretical — they affect real training runs every day.

1. Local Minima

The cost surface of a complex model (like a neural network) is not a simple bowl — it has many valleys (local minima) and a single deepest valley (global minimum). Gradient descent always moves downhill from its starting point, so it may settle into a local minimum and never find the global one.

Two starting points (red dots) leading to different minima. Starting on the left reaches the deeper global minimum.

Good news for deep learning: Research has shown that local minima are less of a problem in practice than previously feared. In high-dimensional spaces (millions of parameters), most local minima have similar cost values to the global minimum. The bigger concern is saddle points.

2. Saddle Points

A saddle point is a point where the gradient is zero but it's neither a maximum nor a minimum — it's a minimum in some dimensions and a maximum in others. In high-dimensional spaces, saddle points are far more common than local minima. Gradient descent can slow to a near-halt near saddle points.

3. Vanishing / Exploding Gradients

In deep neural networks with many layers, gradients are multiplied together as they propagate backwards through the network. If these values are all slightly less than 1, gradients vanish — becoming so tiny that early layers learn almost nothing. If they're all greater than 1, gradients explode — growing so large that training diverges.

Vanishing Gradient

Gradient → 0 in early layers. Symptoms: loss stops decreasing, early-layer weights don't change. Solutions: ReLU activations, batch normalisation, residual connections.

Exploding Gradient

Gradient → ∞ in early layers. Symptoms: loss becomes NaN, parameters blow up. Solutions: gradient clipping, smaller learning rate, weight initialisation.

4. When to Stop — Convergence Criteria

How do we know when gradient descent has finished? Three common stopping rules:

①Fixed number of iterations: Run for exactly N steps (e.g. 1000 epochs). Simple and predictable but may under- or over-train.
②Cost change threshold: Stop when |J(t) − J(t−1)| < ε for some tiny ε (e.g. 10⁻⁶). Cost is no longer decreasing meaningfully.
③Gradient magnitude: Stop when ‖∇J‖ < ε. The gradient is essentially zero — we're at a (hopefully) flat minimum.

5. Feature Scaling

If input features have very different scales (e.g., age in range 0–100, income in range 0–100,000), the cost surface becomes an elongated ellipse rather than a circle. Gradient descent takes an inefficient zigzag path and converges very slowly. Always normalise or standardise your features before running gradient descent.

Standardisation: x' = (x − μ) / σ    →    mean 0, std 1
